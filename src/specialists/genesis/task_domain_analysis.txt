TASK: DOMAIN ANALYSIS
=====================

INSTRUCTIONS
============
Analyze the user's query to determine:

1. Problem Domain: What field/area does this problem belong to?
   - Examples: "Software Engineering", "Legal", "Medical Research", "Business Strategy"
   - Be specific but not overly narrow

2. Complexity Level: How complex is this problem?
   - simple: Single-domain, straightforward task
   - medium: Multi-step, requires coordination between 2-3 specialties
   - complex: Multi-domain, requires 4+ specialists, has dependencies

3. Required Roles: What specialized agents are needed?
   - List specific roles (e.g., "Backend Developer", "Database Administrator", "Legal Analyst", "UX Designer")
   - Focus on distinct expertise areas
   - Avoid redundancy (don't spawn 3 "Backend Developers")

4. Reasoning: Explain your analysis
   - Why this domain?
   - Why this complexity level?
   - Why these specific roles?

5. Tools per Role: Which roles need external tools?
   AVAILABLE TOOLS (only these exist):
   - "web_search"     → Search the web via SearXNG (real-time information, current events, research)
   - "scrape_article" → Extract full content from a URL (detailed reading of specific pages)

   Assign tools ONLY when the role genuinely requires real-world, up-to-date, or external information.
   Leave tools empty ([]) for roles that work purely from reasoning, existing KG context, or internal knowledge.

   Examples of roles that NEED tools:
   - "Market Research Analyst" → ["web_search", "scrape_article"]
   - "Competitive Intelligence Specialist" → ["web_search"]
   - "News & Trends Analyst" → ["web_search", "scrape_article"]

   Examples of roles that do NOT need tools:
   - "Systems Architect" → []  (designs from requirements, no search needed)
   - "Legal Compliance Reviewer" → []  (reasons from domain knowledge)
   - "Technical Writer" → []  (synthesizes from KG context)

OUTPUT FORMAT
=============
Respond with valid JSON only (no markdown, no extra text):

{{
  "domain": "The identified domain",
  "complexity": "simple" or "medium" or "complex",
  "required_roles": ["Role1", "Role2", "Role3"],
  "tools_per_role": {{"Role1": ["web_search"], "Role2": []}},
  "reasoning": "Detailed explanation of your analysis"
}}
