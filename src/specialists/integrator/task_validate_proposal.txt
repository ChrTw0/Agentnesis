TASK: VALIDATE PROPOSAL
=======================

INSTRUCTIONS
============
Validate the agent's proposal against the current Knowledge Graph.

VALIDATION STEPS:

1. PARSE PROPOSAL
   - Read the agent's content (textual description)
   - Read the structured_data (entities and relations to add/update)

2. CHECK FOR CONFLICTS
   Scan for 4 conflict types:

   A. TECHNICAL CONFLICTS
      - Technology/version mismatches
      - Incompatible tools/frameworks
      Example: Proposes a technology incompatible with one already committed in the KG

   B. SEMANTIC CONFLICTS
      - Logical contradictions
      - Inconsistent definitions
      Example: Claims "stateless API" but KG shows "API uses sessions"

   C. TEMPORAL CONFLICTS
      - Dependencies not met
      - Wrong order of operations
      Example: Proposes deployment but testing not complete

   D. SCOPE CONFLICTS
      - Agent working outside expertise
      - Modifying unrelated domains
      Example: Backend agent changing UI components

3. EVALUATE SEVERITY
   - critical: Blocks all progress, must fix
   - high: Major issue, should fix before approval
   - medium: Minor issue, can approve with warning
   - low: Nitpick, informational only

4. CHECK CONFIGURATION
   - strict_mode=true: Reject on medium+ conflicts
   - strict_mode=false: Approve if only low/medium conflicts
   - allow_property_override: Can agent update existing entity properties?

5. MAKE DECISION
   Two outcomes:

   A. APPROVED
      - No critical/high conflicts (or only low conflicts in normal mode)
      - Proposal aligns with Knowledge Graph
      - Provide encouraging feedback
      - List entities/relations to commit

   B. REJECTED
      - Critical or high conflicts detected
      - Provide specific, actionable feedback
      - List ALL conflicts found (up to max_conflicts_per_proposal)
      - Suggest concrete fixes

FEEDBACK QUALITY:
- Be specific: "Change [proposed technology] to [committed technology] (already committed in iteration N)"
- Be actionable: "Add entity 'test_suite' before proposing deployment"
- Be constructive: Don't just say "wrong", explain WHY and HOW to fix

OUTPUT FORMAT
=============
Respond with valid JSON only (no markdown, no extra text):

{{
  "agent_name": "The agent being validated",
  "is_valid": true or false,
  "status": "approved" or "rejected",
  "feedback": "Detailed feedback for the agent",
  "conflicts": ["conflict1", "conflict2"],
  "confidence": 0.0 to 1.0
}}
